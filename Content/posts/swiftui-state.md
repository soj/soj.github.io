# Управление состоянием в SwiftUI

Один из первых вопросов в SwiftUI — где хранить состояние. Короткая шпаргалка,
которой пользуюсь сам:

- **`@State`** — локальное состояние, принадлежащее самому view (переключатель,
  введённый текст). Источник истины живёт внутри view.
- **`@Binding`** — ссылка на состояние, которым владеет родитель. Дочерний view
  читает и меняет, но не хранит.
- **`@Observable`** (макрос из Observation) — для моделей с несколькими полями.
  View перерисовывается только при изменении тех свойств, которые он реально
  читает.
- **`@Environment`** — для зависимостей, которые нужны многим экранам сразу:
  тема, сервисы, текущий пользователь.

Главное правило: у каждого кусочка состояния должен быть ровно один владелец.
Если состояние нужно двум view — поднимаем его к общему родителю и раздаём
вниз через `@Binding`.
