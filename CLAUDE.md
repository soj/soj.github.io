# SergeyMingalev — сайт-визитка

Персональный сайт-визитка iOS / Swift-разработчика: главная + блог + портфолио.
Двуязычный (RU + EN). Статический сайт, собирается генератором **Publish**
(от JohnSundell). Весь сайт — это Swift-программа: контент в Markdown, оформление
описано кодом.

## Стек

- **Publish** — движок статической генерации (читает `Content/`, прогоняет через тему, пишет `Output/`).
- **Plot** — описание HTML/вёрстки темы на Swift.
- **Splash** — подсветка синтаксиса Swift в блоках кода (плагин в пайплайне).
- Исходники самого движка Publish склонированы рядом: `../Publish` (для работы сайта не нужны).

## Как это устроено: что где править

| Хочу… | Файлы | Язык |
|-------|-------|------|
| Добавить/изменить статью, проект, текст | `Content/**.md` | Markdown |
| Имя сайта, секции, язык, контакты, плагины | `Sources/SergeyMingalev/main.swift` | Swift |
| Внешний вид / поведение (меню, карточки, цвета, переключатель языков, подсветка) | `Sources/SergeyMingalev/Theme+Card.swift` | Swift |

**Контент — в MD, оформление — в Swift, Publish склеивает их в HTML.**
Если правка про «как выглядит» — это тема (Swift), а не текст статьи.

## Команды

```bash
publish generate     # пересобрать сайт в Output/
publish run          # пересборка + локальный сервер :8000 (интерактивна — запускать в своём терминале)
```

Превью статики без интерактивности:
`python3 -m http.server 8000 --directory Output`

## Деплой (GitHub Pages)

Сайт развёрнут вживую: **https://soj.github.io/** (RU) и **/en/** (EN).
Репозиторий `soj/soj.github.io`, ветка `main` — исходники; источник Pages = **GitHub Actions**.

Цикл обновления:
1. Правите контент/тему → `git commit`.
2. `git push` в `main`.
3. Workflow `.github/workflows/deploy.yml` собирает сайт (`swift run SergeyMingalev`)
   на macOS-раннере и деплоит `Output/` в Pages.

`Output/` в git не хранится — его генерирует CI. Движок `Publish/` в репозиторий не входит.

## Структура контента

```
Content/
  index.md           → /            (главная RU, текст «обо мне»)
  posts/             → /posts/      (блог RU — секция)
  projects/          → /projects/   (портфолио RU — секция)
  en.md              → /en/         (главная EN)
  en/posts.md        → /en/posts/   (листинг блога EN)
  en/posts/*.md      → /en/posts/<slug>/
  en/projects.md     → /en/projects/
  en/projects/*.md   → /en/projects/<slug>/
```

## Мультиязычность (RU + EN) — сделана вручную

«Вариант A»: RU в корне `/`, EN под `/en/`, переключатель RU↔EN в шапке.
Подробности и нюансы — в файле памяти `bilingual-site.md`. Кратко:

- RU — обычные секции Publish; EN — свободные `Page` (`Content/en/...`).
- Листинги EN строятся в теме из `context.pages`.
- **Индексы EN — файлами рядом с папкой** (`en.md`, `en/posts.md`), НЕ `index.md` внутри
  (иначе путь ломается на `/en/posts/index/`). `path:` во front matter для Page не работает.
- **Слуги (имена файлов) в RU и EN держать одинаковыми** — иначе переключатель ведёт на 404.

### Добавить двуязычную статью
Создать два файла с одинаковым именем:
`Content/posts/my-post.md` (RU) и `Content/en/posts/my-post.md` (EN), затем `publish generate`.

## Известные ограничения

- RSS — только для RU-секций; EN-страницы в ленту не попадают.
- У EN-проектов нет типизированных метаданных (`tech`/`link`) — они в теле markdown; EN-карточки проще RU.
- Нет перевода → переключатель ведёт на 404 (фолбэк пока не сделан).
- В `main.swift` контакты — плейсхолдеры (`you@example.com`, `your-handle`), заменить на реальные.

## Конвенции контента

- Заголовок статьи — первой строкой `# Заголовок` (Publish берёт его в `title` и оставляет в теле;
  тема срезает дубль через `bodyAfterTitle`).
- Front matter: `date`, `description`, `tags`; для RU-проектов ещё `tech` и `link`.
- Блок ` ```swift ` подсвечивается автоматически (пометка `no-highlight` отключает).
